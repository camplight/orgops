### Changed

- Improve bundled setup shell execution by running dependency install and workspace builds in the extracted root directly.

## Release process

Release notes are captured in the GitHub Releases changelog that is generated by
`.github/workflows/release-main.yml` for each release tag in the format
`<semver>-<yyyy-mm-dd>`.

Each release publishes:

- Release notes with commit summaries since the previous release tag
- A `CHANGELOG-<release-tag>.md` artifact attached to the release assets
